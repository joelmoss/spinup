Design Principles

  • Errors surface immediately — red border, restart button appears, crash count visible
  • Healthy = quiet — green dot, no extra attention needed
  • Idle = minimal — gray, recedes visually
  • Inline actions — restart a crashed process without switching windows
  • Focus button — brings that VS Code window to front
  • Everything flat — no expand/collapse, no drilling down. All state visible always.
